OAuth 2.0 Overview

OAuth 2.0 lets an application access protected resources with delegated permission, without asking the user to share their password with that application.

Explanation

Authorization Code with PKCE

A client sends the user to the authorization server. After the user signs in and consents, the server returns an authorization code to the client redirect URI. The client exchanges that code, plus its PKCE verifier, for tokens.

Explanation

Roles and Tokens

  • Resource owner: the user or system that grants access.
  • Client: the app requesting delegated access.
  • Authorization server: authenticates the resource owner and issues tokens.
  • Resource server: hosts the protected API and validates access tokens.

Key idea

An access token represents permission to call an API. It is not the user's password and should be scoped, short-lived, and sent only to the intended resource server.
